# User Stories

## Overview
This directory contains user stories for the Airbnb Clone Backend project, derived from the use case diagram.

## Files
- `user-stories.md` - Complete set of 20 user stories with acceptance criteria

## User Story Categories

### Guest User Stories (7)
Stories focused on guest interactions with the platform:
- User Registration
- Browse Properties
- Search and Filter
- Book Property
- Make Payment
- View Booking History
- Write Reviews

### Host User Stories (5)
Stories focused on host property management:
- List Property
- Manage Availability
- Respond to Bookings
- Receive Payments
- Set Cancellation Policy

### Communication Stories (2)
Stories about user interactions:
- Send/Receive Messages
- Provide Feedback

### Profile & Security Stories (3)
Stories about user accounts:
- Manage Profile
- Verify Identity
- Social Login

### Search & Discovery Stories (2)
Stories about finding properties:
- Apply Filters
- View Property Details

### Additional Stories (1)
- Track Check-in/Check-out

## Story Format
Each user story follows the standard format:

**As a** [user type]  
**I want to** [action]  
**So that** [benefit]

All stories include:
- Clear acceptance criteria
- Related use cases from diagram
- Business rules

## Priority Levels
- **Must Have (MVP)**: 5 core stories
- **Should Have (Phase 2)**: 5 important stories  
- **Nice to Have (Phase 3)**: 10 enhancement stories

## Usage
These user stories serve as the foundation for:
- Sprint planning
- Development tasks
- Testing scenarios
- Product backlog
